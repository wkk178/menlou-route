## 📖 项目背景 
北京西四北三条胡同蕴藏着深厚的历史底蕴与226个院落的千年记忆。然而，传统胡同旅游往往面临“有门楼无解读、有院落无故事、有空间无运营、有文化无活化”的四大痛点。

本项目“门楼层记·故事里”应运而生。我们不仅提出了一套闭环的文化体验体系，更配套开发了本套智慧导览模型。系统通过智能算法、空间地理推演与动态权重模型，为游客量身定制胡同漫步路线，实现数字技术对实体文商的精准反哺。

## ✨ 核心技术亮点 
### 1. 🎯 智能路线推荐算法 
- 多维偏好计算：基于游客的体验偏好（偏好文化或商业）、游览时间、同行人员画像等进行多维雷达打分。
- 空间距离与时序优化：采用 Haversine 算法计算地球球面两点间真实物理距离，结合各点位建议停留时间，推演最优的“由东向西”不走回头路的步行序列。
- 动态权重自反馈：系统具备低成本的“自学习”机制。算法会自动分析近期游客的问卷倾向，并动态更新后台模型权重，实现“越用越懂游客”。

### 2. 🗺️ 高精度实景地图打桩 
- 突破通用地图接口限制：为解决常规商用地图 API 在复杂狭窄胡同中的“穿墙”与坐标漂移问题，本项目采用全手工像素级物理坐标打桩技术。
- 精准咬合真实街区：依托高德地图（Amap）GCJ-02 坐标系，将隆长寺、程砚秋故居等14个核心节点，精确到小数点后五位，完美还原胡同真实倾斜走向，标记分毫不差。

### 3. 🎨 沉浸式交互前端 
- 结合 GSAP 引擎实现具有文化厚重感的丝滑页面滚动与入场动画。
- 响应式设计完美适配移动端边走边看，并定制差异化地图标记，直观呈现业态分布。

## 🛠️ 技术栈 
- 前端交互：HTML5, CSS3, Vanilla JavaScript, GSAP Animations
- 地图渲染：Leaflet.js + 高德地图瓦片服务 (Amap Tiles)
- 算法引擎：Python 3.x, Flask RESTful API
- 数据驱动：无服务器轻量级 JSON 动态存取

---

## 📊 数据洞察模型 
系统内置了针对评委展示与后期复盘的数据监视大屏接口。在系统实际部署运行期间，可实时调取并可视化当前系统学习到的权重参数、历史游客画像以及热门点位分布，直观展现项目的落地运营成果。
